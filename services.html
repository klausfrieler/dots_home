Das Hauptaufgabengebiet von DOTS ist das Hosting von individuell zusamengestellten Testbatterien. Ein weiteres Angebot ist die programmatischer Umsetzung neuer Tests mit dem psychTestR-Framework, dies kann auch die Re-Implementierung historischer Test umfassen, was zu Lehrzwecken von Interesse ist.

Einrichtung einer Testbatterie

Um eine Testbatterie auf DOTS laufen zu lassen, genügt es, sich eine Batterie aus den verfügbaren Tests zusammenzustellen und die Wünsche den Einrichtern mittzuteilen. Diese werden in Absprachee die Testbatterie aufsetzen und freischalten. Die Laufzeit kann individuell vereinbart werden. Dieser Service ist kostenfrei für DGM-Mitglieder, allerdings behält sich die DGM vor, bei sehr hohem Teilnehmeraufkommen ggf. einen anteiligen Unkostenbeitrag zu fordern. Nicht-Mitglieder sprechen uns bitte an, sicherlich lässt sich eine Lösung finden. Die Details werden jeweils individuell vereinbart. Bei Studien mit bis zu mehreren 100 Teilnehmer sollte dies aber kein Problem sei. (Der DOTS läuft zunächst bis zur nächsten Mitgliederversammlung der DGM als Pilotprojekt, d.h. diese Konditionen können sich evtl. noch ändern.)

Oft ist eine Integration einer psychTestR-Testbatterie mit individuellen Fragebögen gewünscht. Wenn diese Fragebögen nicht von allgemeinen Interesse sind, lohnt sich keine eigene psychTestR Implementierung. In diesem Fall kann eine Batterie über spezielle Links auch mit einem Fragebogen auf einer der gängigen Plattformen verknüpft werden. Auch dabei beraten wir Sie gern.

Falls Sie also DGM-Mitglied sind und eine Testbatterie laufen lassen wollen, wenden Sie einfach per Mail an Klaus Frieler .

Keine Kosten! (für DGM-Mitglieder).

Erstellung eines psychTestR-basierten Tests oder Fragebogen

Wenn Sie einen eigenen Hörtest oder Fragebogen entwickeln, kann es für Sie interessant sein, eine psychTestR-Implementation zu erstellen. Dies könnten Sie selber machen, oder, falls das zu viel Aufwand ist oder aus anderen Gründen nicht möglich ist, können Sie sich auch an uns wenden. Wir haben ein Team von erfahrenen Entwicklern, die gerne die Implementation in enger Zusammenarbeit mit Ihnen übernehmen. Die Vorteile einer solchen Umsetzung liegen auf der Hand:

  • Offizielle Referenzimplementation, z.B. als Begleitung für ein Paper
  • Freie Zurverfügungstellung Ihres Testinstruments für interessierte Forscher:innen
  • Beitrag zum DOTS-Testpool
  • Unterstützung von Open Science
  • Größere Sicherheit, dass Ihr Testinstrument von anderen korrekt angewandt wird
  • Ruhm und Ehre

Falls Sie überlegen, eine solche Referenziplementation von uns entwicklen zu lassen, kontaktieren Sie einfach unseren Entwicklungsleiter Klaus Frieler, und wir kommen ins Gespräch. Aufgrund des relative hohen Aufwands einer solchen Implementation, kann dieser Service leider nicht kostenfrei angeboten werden, planen Sie also ein entsprechendes Budget in ihren nächsten Forschungsantrag mit ein.

Kosten nach Vereinbarung.

Hosting von Shiny-Apps

Da es sich um einen Shiny-Server handelt, können nicht nur Tests gehostet werden, sondern auch andere Formen von Shiny-Apps, wie zum Beispiel zur Konfiguration von komplexen Tests (z.B. der HALT Configurator, oder der Gold-MSI Configurator) oder auch zur Datenauswertung und -visualisierung. Falls Sie eine Shiny-App hosten lassen wollen oder Hilfe bei der Entwicklung gewünscht ist, sprechen Sie uns einfach an.

Hosting: Kostenfrei für DGM-Mitglieder
Entwicklung von Shiny-Apps: Kosten nach Vereinbarung.
