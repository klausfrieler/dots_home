Allgemeines

Das Hauptaufgabengebiet von DOTS ist das Hosting von individuell zusamengestellten Testbatterien. Ein weiteres Angebot ist die programmatischer Umsetzung neuer Tests mit dem psychTestR, dies kann auch die Re-Implementierung historischer Test umfassen, was zu Lehrzwecken von Interesse ist.

Einrichtung einer Testbatterie

Um eine Testbatterie auf DOTS laufen zu lassen, genügt es, sich eine Batterie aus den verfügbaren Tests zusammen zu stellen und die Wünsche der Einrichter (Klaus Frieler) mittzuteilen. Dieser wird in Absprache die Batterie aufsetzen und freischalten. Die Laufzeit kann individuell vereinbart werden. Dieser Service ist kostenfrei, allerdings behält sich die DGM vor, bei sehr hohem Teilnehmeraufkommen ggf. einen anteiligen Unkostenbeitrag zu fordern. Die Details werden individuell vereinbart. Bei Studien mit bis zu mehrern 100 Teilnehmer sollte dies aber kein Problem sei. (Der DOTS läuft zunächst bis zur nächsten Mitgliederversammlung der DGM als Pilotprojekt, d.h. diese Konditionen können sich evtl. später noch ändern.)

Oft ist eine Integration einer psychTestR-Testbatterie mit individuellen Fragebögen gewünscht. Wenn diese Fragebögen nicht von allgemeinen Interesse sind, lohnt sich keine eigene psychTestR. In diesem Fall kann eine Batterie über spezielle Links auch mit einem Fragebogen auf einer der ghöngigen Plattformen verknüpft werden. Auch dabei beraten wir Sie gern.

Falls Sie also DGM-Mitglied sind und eine Testbatterie laufen lassen wollen, wenden Sie einfach per Mail an Klaus Frieler .

Keine Kosten! (für DGM-Mitglieder).

Erstellung einer psychTestR-basierten Tests oder Fragebogen

Wenn Sie einen eigenen Hörtest oder Fragebogen entwickeln, kann es für Sie interessant sein, eine psychTestR-Implementation zu erstellen. Dies könnten Sie selber machen, oder, falls das zu viel Aufwand ist oder aus anderen Gründen nicht möglich, können Sie sich auch an uns wenden. Wir haben ein Team von erfahrenen Entwicklern, die gerne die Implementation in enger Zusammenarbeit mit Ihnen übernehmen. Die Vorteile einer solchen Umsetzung liegen auf der Hand:

  • Offizielle Referenzimplementation, z.B. als Begleitung für ein Paper
  • Freie ZUrverfügungstellung Ihres Testinstruments für interessierte Forscher:innen
  • Beitrag zum DOTS Test Pool
  • Unterstützung von OpenScience und Testingmethoden
  • Größere Sicherheit, dass Ihr Testinstrument von anderen korrekt angewandt wird
  • Ruhm und Ehre

Falls Sie überlegen, eine solche referenzerstellung von uns entwicklen zu lassen, kontaktieren Sie einfach unseren Entwicklungsleiter Klaus Frieler, und wir kommen ins Gespräch.
